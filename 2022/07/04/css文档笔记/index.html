css文档笔记

层叠样式表–是一门样式表语言
用于设计风格和布局

type=”text/css”–它们是告诉浏览器这里面的文本内容(text)要当层叠样式表 (css)来解析

css注释
/* */

一切皆盒子

清除默认样式

css由许多模块构成，许多模块都被组织在自己单独的文档页面
比如：background-x

css规范

在html中添加css

【无障碍】：意思是网页能让每一个访客都能理解使用

  1. 外部样式表
    将 CSS 编写在扩展名为.css 的单独文件中，并从 HTML link 元素引用它的情况：

    link rel=”stylesheet样式表” href=”styles.css”

  2. 内部样式表
    将 CSS 放在 HTML 文件head标签里的style标签之中。

  3. 内联样式
    内联样式表存在于 HTML 元素的 style 属性之中。其特点是每个 CSS 表只影响一个元素
    【最好不要这样做】

在 css 中，属性和值都是区分大小写的。每对中的属性和值由冒号 (：) 分隔。

函数
calc()
–简单的计算

rotate()–旋转
transform: rotate(0.8turn)

@rules规则
–为css提供一些关于如何表现的知道

@import–引入css样式表

@media–媒体查询

用处：
根据设备的类型【打印机/带屏幕的折百】或者特定的特征和设备参数【屏幕分辨率吧/浏览器视窗宽度】修改网站/应用程序

目的：

  1. 有条件的通过 @media 和 @import at-rules 用CSS 装饰样式。
  2. 用media= 属性为style,link, source和其他HTML元素指定特定的媒体类型

css是如何工作的？

  1. 浏览器载入html文件
  2. 将html文件转化为一个DOM【文件在计算机内存中的表现形式】
  3. 浏览器拉区HTML相关资源【图片/视频/css样式/js】
  4. 解析css【根据选择器的不同类型分到不同的桶中】–浏览器基于它找到的不同的选择器，将不同的规则应用在对应的DOM节点中，并添加节点依赖的样式—【渲染树】
  5. 渲染树根据应该出现的结构进行布局
  6. 网页展示在屏幕上–【着色】

DOM
一个DOM有一个树形结构，标记语言中的每一个元素/属性/文字都对应着结构树中的一个节点

initial：浏览器默认样式
unset：自然值

重设所有属性值
all：unset

盒模型

分类：
块级盒子 block box
【占据该方向上的所有可用空间/每个盒子都会换行/width，height可以发挥作用/
内边距（padding）, 外边距（margin）和 边框（border）会将其他元素从当前盒子周围“推开”】

块级元素：
标题，段落

内联盒子 inline box
【盒子不会换行/width，height不起作用/垂直方向的内边距、外边距以及边框会被应用但是不会把其他处于 inline 状态的盒子推开。/水平方向的内边距、外边距以及边框会被应用且会把其他处于 inline 状态的盒子推开。

行内元素
a/span/em/strong

元素之间的切换
display：inline/bolck

改变内部显示类型
外部和内部显示类型

display属性
【改变盒子的外部显示类型是块级还是内联，这将会改变它与布局中其他元素的显示方式】

如果设置 display: flex，在一个元素上，外部显示类型是 block，但是内部显示类型修改为 flex

正常文档流

盒模型
标准盒模型
在标准模型中，如果你给盒设置 width 和 height，实际设置的是 content box。
padding 和 border 再加上设置的宽高一起决定整个盒子的大小

盒子宽度=width+paddingx2+borderx2

【margin不计入实际大小，但会影响到盒子外部空间】

IE盒模型
使用这个模型，所有宽度都是可见宽度，所以内容宽度是该宽度减去边框和填充部分

width = content+padding+border

【使用】
box-sizing: border-box

外边距折叠
有两个外边距相接的元素，将外边距合并为一个外边距

display:inline-block
在内联和块之中提供了一个中间状态
【达到块级的部分效果】

设置width 和height 属性会生效。
padding, margin, 以及border 会推开其他元素。

但不会跳转到新行

背景和边框

背景颜色 background-color
背景图片 background-image
【大图不会缩小/小图则会平铺】
控制：
background-repeat

no-repeat — 不重复。
repeat-x —水平重复。
repeat-y —垂直重复。
repeat — 在两个方向重复。

设置图片大小
background-size：长度/百分比

关键字
cover:使图片足够大，覆盖盒子区，仍然保持其宽高比

contain：使图片的大小适合盒子里【可能有间隙】

背景图像定位
background-position: top center; background-position: 20px 10%;
【可以混合使用方向/百分比/大小】

渐变背景
linear-gradient

多个背景图像
background-image: url(image1.png), url(image2.png), url(image3.png), url(image4.png);
background-repeat: no-repeat, repeat-x, repeat;
background-position: 10px 20px, top right;

背景滚动
background-attachment

scroll：背景在页面滚动时滚动
fixed：背景固定在视图端口上，页面滚动，背景不会滚动
local：滚动元素时，背景也随之滚动

溢出的内容

overflow：控制元素溢出

默认值：visible–内容溢出时，可以看到

hidden：隐藏掉【确定隐藏内容不会引起问题】

scroll：滚动条
overflow-y: scroll【仅在y方向滚动】

auto：想让滚动条在有比盒子所能装下更多的内容的时候才显示【由浏览器决定是否显示滚动条】

溢出建立以块级排版上下文BFC
，在你使用诸如scroll或者auto的时候，你就建立了一个块级排版上下文

css的值和单位

数字，长度，百分比

绝对长度–与其他任何东西无关
px

相对长度–视图端口的大小
em【font-size相对于父元素的字体大小】
ex：字符x的高度
ch：数字0的高度
rem：根元素的字体大小
lh：元素的line-height
vw：视图宽度的1%【更改浏览器窗口的尺寸，框的大小会更改】
vh：视图高度的1%
vmin：视图较小尺寸的1%

em/rem
作用：从框到文本调整大小

百分比
相对于其他值设置的

数组
透明度
.box {
opacity: 0.6;
}

颜色
十六进制RGB值 #02798b

RGB函数–红绿蓝 rgb(2, 121, 139)

RGBA–红绿蓝不透明度 rgb(2, 121, 139)

HSL 色调，饱和度 亮度值 hsl(188, 97%, 28%)

在css中调整大小【可响应】

最大最小尺寸
min-height: 150px;
盒子就会一直保持大于这个最小高度，但是如果有比这个盒子在最小高度状态下所能容纳的更多内容，那么盒子就会变大。

max-width：
在没有足够空间以原有宽度展示图像时，让图像缩小，同时确保它们不会比这一宽度大。
max-width: 100%，那么图像可以变得比原始尺寸更小，但是不会大于原始尺寸的 100%。

视口单位

vh/vw–随用户的视口改变大小

让你的页面的那个部分有 100vh 高的话，会把剩下的内容推到视口的下面，只有向下滚动文档的时候它们才会出现。

CSS布局

正常布局流

浏览器默认的html布局方式

覆盖默认布局

  1. display
    正常布局流中的内容都有一个display的默认值
    display：block
    display：inline

    别的值
    inline-block
    flex
    grid

  2. 浮动

  3. position

  4. 表格布局

  5. 多列布局

弹性盒子

display：flex–div拍成一行–变成了flex项

设置 flex：1–所有子元素伸展并填充容器
.wrapper > div {
flex: 1;
}

grid布局

.wrapper {
display: grid;
grid-template-columns: 1fr 1fr 1fr;—定义列轨道
grid-template-rows: 100px 100px;–定义行轨道
grid-gap: 10px;
}

grid-column：指定从哪一行开始，哪一列结束

.box1 {
grid-column: 2 / 4;
grid-row: 1;
}

浮动

改变元素本身和在正常布局流中跟随他的其他元素—会在正常布局流中移除–其他元素会环绕

float：

left — 将元素浮动到左侧。
right — 将元素浮动到右侧。
none — 默认值，不浮动。
inherit — 继承父元素的浮动属性。

定位

把从正常布局流中应该在的位置移动到另一个位置
【微调页面一个特殊项】

  1. 静态定位：static
  2. 相对定位：relative【两个元素叠放在页面上】
  3. 绝对定位：absolute【单独放在一个图层】
  4. 固定定位：fixed【将元素相对浏览器视口固定，而不是另一个元素】—导航菜单
