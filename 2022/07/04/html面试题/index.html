HTML面试题

1.说说对HTML语义化的理解

是什么
我们在写html结构时用有英文语义的标签，更有利于开发人员和机器的阅读和理解

语义元素
form-table-nav-header

非语义元素
div-span

为什么要语义化

  1. 没有css，页面能更好的呈现内容和结构，容易阅读
  2. 提升用户体验：title/alt属性用于名词解释，label标签的灵活运用
  3. 有利于SEO优化：有助于爬虫抓取更多的有效信息–爬虫依赖于标签来确定上下文和各个关键字的权重
  4. 可访问性：有利于屏幕阅读器
  5. 国际化：统一标准
  6. 可维护性：减少网页的差异

应该注意些什么问题

  1. 尽量少使用div/span
  2. 在语义不明确的时候，可以使用div和p的时候，尽量用p–因为默认情况下有上下间距，有利于兼容特殊终端
  3. 不要使用纯样式标签：b/u—css
  4. 强调–strong/em【不要用b/i】
  5. 使用表格，标题用caption，表头用thead

3.DOCTYPE是干嘛的，都有哪些属性

document type文档类型的缩写

作用
告诉浏览器的解析器使用HTML规范或者XHTML规范来解析页面

目的
防止浏览器在渲染文档时，进入怪异模式渲染，确保浏览器按照标准模式进行渲染

现在使用的就是html5中的声明，只有一种
!DOCTYPE html

浏览器的排版引擎使用三种模式

  1. 怪异模式【混杂/兼容模式】：排版会模拟出非标准行为，会产生一定差异
  2. 接近标准模式：只有少数的怪异行为被实现
  3. 标准模式：w3c标准

不写或前面有其他内容的影响

  1. 浏览器无法获知HTML/XHTML的文档类型，就会使浏览器进入怪异模式渲染网页，不同浏览器显示效果就会不一致

  2. 如果有任何其他字符位于 DOCTYPE 之前，比如注释或 XML 声明，会导致 Internet Explorer 9 或更早期的浏览器触发怪异模式。

XHTML
可扩展的超文本标记语言–增强的html

DOCTYPE对HTML标签的影响
有一些html标签在升级中被废除了，有一些增加了，这决定了哪些html标签可以合法使用

4.meta标签干什么的，都有什么属性和作用

是什么
用于描述网页的元信息，不会显示在页面上，但是对于机器是可读的，并告诉机器如何解析这个页面

如：浏览器（如何显示内容或重新加载页面），SEO 优化（keywords、description、robots）其他 web 服务等

常用属性

  1. charset：规定html页面的字符编码
  2. content：当meta标签有http-equiv/name属性，一定要有content属性对其说明
  3. http-equiv：添加http头部内容–对于一些自定义的，需要额外添加的http头部内容，需要发送到浏览器中
  4. name：元数据名称

7.说一下href和src的区别

href–指向网络资源的位置
建立当前页面和引用资源之间的关系
页面会并行加载后续内容【如果我们在文档中添加link href=”common.css” rel=”stylesheet”那么浏览器会识别该文档为 css 文件，就会并行下载资源并且不会停止对当前文档的处理。这也是为什么建议使用 link 方式来加载 css，而不是使用@import 方式】
src–指向外部资源的位置
替换当前元素
浏览器需要加载完毕src的内容才能继续往下走【这也是为什么将 js 脚本放在底部而不是头部。】
