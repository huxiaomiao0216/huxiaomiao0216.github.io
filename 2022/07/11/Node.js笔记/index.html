fs文件系统模块

1.什么是fs文件系统模块？

用来操作文件的模块–提供一系列方法和属性

方法：
fs.readFile()–读取文件中的内容
fs.writeFile()–向指定文件中写入内容

使用：
使用fs模块来操作文件，就需要先导入
const fs = require(‘fs’);

2.fs.readFile()

语法格式：
fs.readFile(path,[options],callback)

path:文件路径
options：以什么编码格式读取
callpack:文件读取完后，可以通过回调函数得到文件内容

判断是否读取成功
err对象是否为null

–判断err是否能转化为true–能–文件读取失败

3.fs.writeFile()

fs.writeFile(file,data,[options],callback)

data:要写入的内容

path路径模块

1.是什么？

处理路径的模块

方法
path.join()–将多个路径片段拼接成一个完整的路径字符串
path.basename()–用来从路径字符串中，将文件名解析出来

使用
const path = require(‘path’)

2.路径拼接

path.join()-[常用]

语法格式
path.join([…paths])
返回值：string

../—–抵消一层路径

path.basename()

获取路径的最后一个部分–获取路径中的文件名

语法格式
path.basename(path,[txt])

path.extname()

获取路径扩展名部分

语法格式
path.extname(path)

HTTP模块

1.什么是HTTP模块？

客户端：消费资源
服务端：对外提供网络资源

作用
http模块是创建web服务器的模块
通过 http.createServer() 方法–对外提供web服务

服务器安装了web服务器软件–Apache

使用
导入
const http = require(‘http’)

2.服务器

IP地址
每台计算机的唯一地址–电话号码

点分十进制 192.168.1.1

域名和域名服务器

端口号
有很多个web服务，每个都对应唯一一个端口号

3.创建web服务器

导入http模块

创建web服务器实例–createServer()

为服务器实例绑定request事件–on

启动服务器–listen

req请求对象
只要服务端收到了客户端的请求，就会调用通过server.on()为服务器绑定的request事件处理函数

【为了访问与客户端相关的数据或属性】

res响应对象
与服务器相关的数据或属性

end–向客户端发送指定的内容，并结束这一次请求

解决中文乱码
res.setHeader(‘Content-Type’,’text/html;charset=utf-8’)

4.根据不同url响应不同html内容

获取请求的url地址

判断用户请求的是否为/或/index.htmlshouted

res.end()–把响应返回

模块化

1.是什么？

将复杂问题，分解
模块是可分解，组合的单元

遵循固定规则， 把大文件拆分成独立并互相依赖的多个小模块

好处
提高代码复用性，可维护性，按需加载

模块化规范
对模块拆分组合时，需要遵循的规则
–使用什么语法格式引用模块
–在模块中使用什么语法格式向外暴露成员

2.Node.js中的模块化

内置模块–fs，path，http
自定义模块–用户创建的每个.js文件
第三方模块–使用前需要下载

3.加载模块

require–按需加载
用户自定义的–里面写文件路径-可以省略文件后缀

4.模块作用域

在自定义模块中的变量，方法等，只能在当前模块被访问–即使导入也不能用

好处–防止全局变量污染问题

5.module对象

向外共享模块作用域中的成员

module对象–每个js自定义模块都i有–存储了与当前模块有关的信息

module.exports–默认等于一个空对象–使用这个将模块内的成员共享出去，供外界使用

在外界导入一个自定义模块是，得到的成员是，通过module.exports的空对象{}

module.exports.username = ‘za’
向对象上挂载一个属性–可以对外暴露

注意点
require()导入模块–导入的结果–永远以module.exports指向的对象为准

6.exports对象

默认情况下：exports和module.exports指向同一个对象

7.模块化规范

node.js遵循了CommonJS规范【规定了模块的特性和各模块之间如何互相依赖】

Common.js规定：

  1. 每个模块内部，module变量代表当前模块
  2. module变量是一个对象，它的exports属性是对外的接口
  3. require()用于加载模块

模块的加载机制

  1. 优先从缓存中加载
    多次require()不会导致模块的代码被执行多次

  2. 内置模块的加载机制
    优先级最高

  3. 自定义模块的加载机制
    require(‘./‘)不许以./或者../开头

  4. 第三方模块加载机制
    require()不是内置模块，也没有以./开头，则会从当前模块的父目录开始，尝试从node_modules文件中加载第三方模块

express

1.初始Express

什么时Express
和node内置的http模块类似—用来创建web服务器的

优点
基于内置http模块进一步封装的，能提升开发效率–第三方模块

能做什么
常见的两种服务器：

  1. web网站服务器：提供web网页资源
  2. API接口服务器：提供API接口的服务器

Express可以快速创建 web网站/API接口 服务器

2.Express的基本使用

安装
npm i express

监听GET请求
app.get()—监听客户端的GET请求

监听post请求
app.post()-监听客户端的post请求

把内容响应给客户端
res.send()–把处理好的内容，发送给客户端

获取URL中携带的查询参数
req.query对象–可以访问到客户端通过查询字符串的形式，发送到服务器的参数

获取动态参数
req.params

3.托管静态资源

express.static()–创建一个静态资源服务器
app.use(express.static(‘public’))–对public下的图片/css文件/js文件对外开放访问

挂载路径前缀
app.use(‘/public’,express.static(‘public’))

4.nodemon

监听项目变化，自动重新启动

路由

1.什么是路由？

路由就是 映射关系

express–客户端的请求与服务器处理函数之间的映射关系
由三部分组成【请求的类型/请求的URL地址/处理函数】
app.method(path,handler)

2.路由的匹配过程

请求到达服务器–路由匹配【从上到下按顺序匹配】–成功后【请求类型和请求的URL同时匹配成功】调用对应的处理函数

3.路由的使用

将路由挂载到app上

4.模块化路由

将路由抽离为单独的模块

实现步骤：
1.创建路由模块对应的.js文件
2.调用express.Router()函数创建路由对象
3.向路由对象上挂载具体的路由
4.使用module.exports向外共享路由对象
5.使用app.use()函数注册路由模块

模块前缀
add.use(‘/api’,userRouter)

5.express中间件

中间处理环节—对请求做预处理

中间件函数的形参列表中，必须包含next参数

next函数的作用：
把流转关系转交给下一个中间件或路由
